// the idea

You'd rather build than plan.

So don't. Write code the way you actually think — one commit at a time, no design doc, no checklist to forget halfway through.

crew rides along. The moment a commit lands, it's reviewed: a correctness audit, a quality pass, an editorial sweep. The rigor arrives continuously, while you stay in flow.

// the crew

Three skills.
One loop.

flow// the loop
Drives build → review for every commit, every chunk, every turn. Calls the rest of the crew for you.
auditor// correctness
A correctness-only review of any diff — logic errors, edge cases, broken contracts, intent drift.
editor// the prose
Strips AI-isms, narration, and conversational residue out of your comments and docs.

// the gate

A second model checks the work.

Most review tools ask one model to grade its own homework. crew doesn't.

After Claude audits and simplifies a change, OpenAI Codex reviews it with fresh eyes — a different model, different blind spots — specifically to catch what the first pass missed. That cross-model gate is the merge bar.

You move fast and loose. What comes out has been audited, simplified, edited, and cross-examined.

// get in flow

Bring your crew.

One prerequisite. crew's cross-model gate runs the real OpenAI Codex CLI. The bridge to it installs with crew, but the CLI is yours to provide — the codex command on your PATH, plus OpenAI auth. Without it, the Codex passes skip and everything else still runs.
